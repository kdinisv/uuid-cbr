# Официальные документы

В этой папке сохранены официальные материалы Банка России, которые относятся к правилам присвоения УИд и используются как reference для проверки реализации.

## Файлы

- `cbr-5251-u.pdf` — Указание Банка России от 09.09.2019 № 5251-У «О правилах присвоения уникального идентификатора договора (сделки), по обязательствам из которого (из которой) формируется кредитная история».
- `cbr-758-p-appendix-2.pdf` — приложение 2 к Положению Банка России № 758-П. На официальной странице ЦБ именно на него дана ссылка как на документ, в соответствии с которым должно выполняться присвоение УИд.
- `cbr-c-generator.zip` — официальный пример исходного кода генератора УИд на языке C, опубликованный Банком России.
- `cbr-algorithm-findings.md` — результаты сверки текущей JS-реализации с официальным примером ЦБ.

## Источник

- Официальный URL ЦБ на момент сохранения: https://www.cbr.ru/queries/unidbquery/file/90134?fileid=931
- Связанная официальная страница по теме: https://www.cbr.ru/ckki/poryadok-formirovaniya-kreditnoy-istorii/
- Официальная ссылка на приложение 2: https://www.cbr.ru/StaticHtml/File/145651/pril2.pdf
- Официальная ссылка на пример генератора на C: https://www.cbr.ru/StaticHtml/File/145651/C.zip

## Где искать алгоритм

- В `cbr-5251-u.pdf` находится нормативная база.
- В `cbr-758-p-appendix-2.pdf` находится официальный документ, на который ЦБ прямо ссылается для правил присвоения УИд.
- В `cbr-c-generator.zip` находится официальный пример реализации генератора, который удобнее всего использовать для технической сверки алгоритма.

## Зачем это в репозитории

- сохранить нормативный reference рядом с кодом;
- сохранить рядом официальный пример алгоритма, а не только нормативный документ;
- не зависеть от переездов ссылок на сайте ЦБ;
- использовать документ при проверке соответствия алгоритма генерации требованиям регулятора.
